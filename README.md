<summary><b>⚙️ CronaFlow — Resilient Distributed Task Scheduler</b></summary>

<br/>

| Category | Details |
|:--|:--|
| **Stack** | Spring Boot, React, Redis, MongoDB, JWT, Docker, Spring WebFlux |
| **Scale** | Stable ingestion throughput of 500+ tasks/sec |
| **Performance** | Sub-1.3s average scheduling latency |
| **Security** | Custom JWT authentication for the monitoring frontend |
| **Impact** | Fault-tolerant DAG scheduling with retries, orphan recovery, DLQ isolation, and real-world async integrations |
| **Repository** | [github.com/kshitij3103/CronaFlow](https://github.com/kshitij3103/CronaFlow) |

CronaFlow is a distributed task scheduler designed for resilient workflow execution. It uses active-passive leader election and Redis distributed locks to prevent duplicate execution across horizontally scaled nodes. The platform supports Directed Acyclic Graph dependencies, configurable retries, automatic orphaned-task recovery, and Dead Letter Queue handling for permanent failures.

The React dashboard provides real-time visibility into queue states and workflow execution, while non-blocking Spring WebFlux workers support SMTP dispatch, Gemini API processing, and GraphQL-based LeetCode scraping.

</details>
